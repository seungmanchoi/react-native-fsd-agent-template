# Ideate — 앱 아이디어 리서치 스킬

앱 아이디어를 체계적으로 리서치하고 실현 가능한 아이디어를 제안하는 스킬.

## Trigger
- "아이디어 줘", "앱 뭐 만들까", "어떤 앱이 좋을까"
- "트렌드 조사", "인기 앱 분석", "시장 분석"

## Steps

### Step 1: 의도 명확화 게이트 (Clarity Gate)

리서치 착수 전, 사용자 입력의 명확도를 평가한다. 입력이 충분히 명확하면
**되묻지 말고 바로 Step 2로 진행**하고, 핵심 축이 비어 모호할 때만 질문한다.

**충분 조건 (아래 6개 축 중 4개 이상이 확정되면 게이트 통과):**
- 해결하려는 문제/페인포인트
- 타겟 사용자
- 카테고리·도메인
- 차별화 방향 단서
- 수익 모델 방향 (광고/구독/IAP)
- 플랫폼·시장 (한국/글로벌)

**먼저 기존 컨텍스트로 축을 채운다:**
- `_workspace/spec.md`가 있으면 `project.context`, `project.category`,
  `deployment.platforms`, `monetization.model`, `ux.languages`와 `*_notes`를
  Read 하여 채울 수 있는 축을 먼저 채운다 (orchestrate Phase 0 안에서 호출된
  경우 같은 질문을 반복하지 않기 위함 — **비어 있는 축만 보충 질문**한다).
- spec.md가 없으면(ideate 단독 실행) 사용자 입력만으로 평가한다.

**명확화 루프 (충분 조건 미달일 때만):**
- 비어 있는 축에 대한 질문 2~4개를 `AskUserQuestion`으로 한 번에 묶어 던진다
  (관심 카테고리 / 타겟 시장 / 개발 규모 / 수익 방향 등)
- 답변이 또 다른 분기를 낳으면 그 시점에만 후속 질문 (**최대 2라운드**)
- 사용자가 "알아서 해 / 자유롭게 제안"이라고 하면 즉시 통과하고 합리적
  기본값(전체 트렌드 조사, 1인 개발, 한국+글로벌)을 사용한다
- `_workspace/spec.md`에 `execution.unattended: true`면 **질문을 생략**하고
  `on_ambiguity` 정책(pick_first / pick_recommended / ask)에 따라 결정한다

**결과 반영:** 수집한 답은 `_workspace/spec.md`의 `project.context` / 해당
`*_notes` 필드에 반영하거나, spec.md가 없으면 Step 2 리서치의 입력 컨텍스트로
사용하여 아이디어 제안을 더 구체적이고 풍부하게 만든다.

### Step 2: 트렌드 리서치
- WebSearch로 최신 앱 트렌드 조사
- App Store/Google Play 인기 차트 카테고리별 분석
- 최근 주목받는 앱 유형, 기술 트렌드 파악
- 사용자 불만/페인포인트가 많은 영역 탐색

### Step 3: 경쟁 앱 분석
- 유사 카테고리 상위 앱 5~10개 분석
- 각 앱의 핵심 기능, 강점, 약점 정리
- 사용자 리뷰에서 반복되는 불만 추출
- 차별화 기회 식별

### Step 4: 아이디어 도출
- 3~5개의 구체적 아이디어 제안
- 각 아이디어에 포함할 내용:
  - 앱 이름 (가안)
  - 한줄 설명
  - 타겟 사용자
  - 핵심 기능 3가지
  - 기존 앱 대비 차별점
  - 수익 모델
  - React Native + Expo 기술적 실현 가능성

### Step 5: 사용자 선택
- 아이디어 목록 제시
- 사용자에게 선택 또는 조합 요청
- 선택된 아이디어를 `_workspace/idea/app-concepts.md`에 저장
