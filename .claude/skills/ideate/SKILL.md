# Ideate — 앱 아이디어 리서치 스킬

앱 아이디어를 체계적으로 리서치하고 실현 가능한 아이디어를 제안하는 스킬.

## Trigger
- "아이디어 줘", "앱 뭐 만들까", "어떤 앱이 좋을까"
- "트렌드 조사", "인기 앱 분석", "시장 분석"

## Steps

### Step 1: 방향 설정
- 사용자에게 관심 카테고리 확인 (없으면 전체 트렌드 조사)
- 타겟 시장 확인 (한국/글로벌)
- 개발 규모 확인 (1인/소규모 팀)

### Step 2: 트렌드 리서치
- WebSearch로 최신 앱 트렌드 조사
- App Store/Google Play 인기 차트 카테고리별 분석
- 최근 주목받는 앱 유형, 기술 트렌드 파악
- 사용자 불만/페인포인트가 많은 영역 탐색

### Step 3: 경쟁 앱 분석
- 유사 카테고리 상위 앱 5~10개 분석
- 각 앱의 핵심 기능, 강점, 약점 정리
- 사용자 리뷰에서 반복되는 불만 추출
- 차별화 기회 식별

### Step 4: 아이디어 도출
- 3~5개의 구체적 아이디어 제안
- 각 아이디어에 포함할 내용:
  - 앱 이름 (가안)
  - 한줄 설명
  - 타겟 사용자
  - 핵심 기능 3가지
  - 기존 앱 대비 차별점
  - 수익 모델
  - React Native + Expo 기술적 실현 가능성

### Step 5: 사용자 선택
- 아이디어 목록 제시
- 사용자에게 선택 또는 조합 요청
- 선택된 아이디어를 `_workspace/01_idea_research.md`에 저장
